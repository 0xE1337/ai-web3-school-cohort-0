## 关键概念 / 笔记（3 条）

### 1. Prompt → Workflow → Agent 的三段边界
- **Prompt** = 让模型回答，决策权在人
- **Workflow** = 路径固定，模型是其中一个节点（可预测、可调试）
- **Agent** = 模型自主规划、跨轮管理状态、动态调用工具（开放、风险也开放）
- 选哪种取决于「任务能否提前写死步骤」与「是否需要跨会话状态」，不是越 agent 越好

### 2. 钱包不是普通账号，签名不是普通登录
- 助记词 / 私钥 / 地址三者一旦泄露不可挽回，因此 AI Agent **绝不能直接接触私钥**
- 签名 = 授权一个**具体动作**（不是"点确认"），授权 ≠ 转账，要看签的是什么 data
- 任何涉及签名、授权、转账、合约写入的步骤，**必须保留人工确认节点**

### 3. AI × Web3 最小交叉实验的五步链路
> AI 生成 → 人工复核 → 钱包确认 → 链上执行 → 区块浏览器验证

这条链路是 Cohort 0 真正的「最小可交付单元」。任何想被算作 AI×Web3 的实践，都应该能在这条链路上跑通一次。把任何一个环节抽掉，要么变成纯 AI 玩具，要么变成纯链上脚本。
